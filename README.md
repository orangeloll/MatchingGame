# Matching Game

같은카드맞추기게임입니다. 2학년 텀프로젝트 과제로 만들었던 프로젝트라 부족한 부분이 많습니다.

## 사용 프로그램

사용 언어 : Java<br>
운영체제 : Window<br>
사용 프로그램 : Eclipse

## 게임의 규칙
세개의 단계로 나뉘어 있는 같은카드맞추기 게임입니다. 카드의 그림은 과일들로 이루어져 있으며 맞춰야 하는 과일들은 위에 글로 나와있습니다. 
화면에 나와있는 카드들을 모두 뒤집으면 끝납니다.

## 문제점 :🥲:
가장 큰 문제점은 이 당시의 제가 주석의 중요성을 제대로 알지 못했다는 점입니다. 주석의 부재로 코드를 살펴볼 때 각 부분이 어느 기능을 하는지 파악하기 어렵습니다.
변수의 이름 또한 성의 없이 이미지일 경우 img, img1, img2 ...라고 지었고 처음 클릭되는 버튼을 담기위한 버튼조차 이름을 firstclick 으로 지었습니다. firstClick 처럼이라도 했으면 조금 알아보기 쉬웠을 것 같아 아쉬움이 남습니다.
주석과 변수의 이름 뿐 아니라 기능적으로도 문제가 많은 코드입니다. 쓸모없는 변수나 코드가 꽤 많습니다. 당시에 왜 이렇게 짠 것인지에 대한 해답을 듣고 싶지만 주석이 없어 파악이 어렵습니다.
기능적 문제는 뒤집힌 짝이 맞을 때는 괜찮지만, 짝이 맞지않을 때가 문제입니다.


https://github.com/orangeloll/MatchingGame/assets/149676286/2fc109f6-4de5-4b74-ae67-5a38116bb60d






두번째로 뒤집은 카드의 앞면이 보이지 않습니다. <br>쉬운 단계의 경우 카드의 수가 적어 2번째로 뒤집은 카드의 면이 보이지 않아도 오랜 시간이 걸리지 않아 괜찮을지 모르지만, 중간 단계부터는 두번째로 뒤집은 카드가 보이지 않는게 게임 시간과 진행 자체에 큰 문제입니다.
고쳐서 코드를 올릴까 했지만 이렇게 오류가 나는 코드를 다시 보기 어려울 것 같아서 일단 깃허브에 업로드 후 고쳐보려고 합니다.
